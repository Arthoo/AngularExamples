Generic lists in Angular

In Angular we can use content projection to insert content inside the template of a component. This is useful in many scenarios, but it does not work when we iterate over a list of items and want to pass the item to the projected component.

The following example shows how to create a list component that can have a custom component for displaying items inside. This can be very useful if you would like to have several lists in your application with the same style and functionalities for search/filter/pagination but with different content. The same technique could be used for other components that iterate over items and you would like to provide a custom component depending on where it is used, e.g. select/drop-downs.

The two lists below are instances of the same list component but uses different components to display the items inside.

In the list component we have an input for the list of item data, and a template reference with the component to be used for displaying each item.

Inside the list template we iterate over the list items and use *ngTemplateOutlet to instantiate the referenced component for each item. By setting the item data in the context we can provide it to the component we instantiate.

When using the list component, we can simply add a nested item component within a template inside the list. The item data can be accessed by assigning a template input variable (let-cat="item" in this example).
